# Plugin Template

Copy this folder to `plugins/sources/<your_source>/` and follow the steps below.
The folder name must match the `source_key` value in `source.json`.

For a complete walkthrough, see `docs/PLUGIN_DEVELOPMENT.md`.

---

## Quick steps

1. Copy this folder: `plugins/sources/<your_source>/`
2. Rename the class in `plugin.py` and implement the three required methods.
3. Fill in `source.json` (see field reference below).
4. Restart `ingest.py` or `app.py` — auto-registration runs on startup.
5. If your source requires credentials, go to `/settings`, enter them, and enable the source.

---

## `source.json` field reference

| Field          | Type            | Description |
|----------------|-----------------|-------------|
| `source_key`   | string          | Unique identifier for the source. **Must match the folder name exactly.** Used as the registry key and as the `source` field in every stored listing. |
| `display_name` | string          | Human-readable name shown in the Settings UI. |
| `description`  | string          | One-sentence description shown below the source name in Settings. |
| `home_url`     | string          | URL of the source's homepage. Shown as a link in the Settings UI. |
| `fields`       | array of objects | List of credential fields the source requires. Use an empty array (`[]`) for keyless (public API) sources. |

### `fields` entry structure

Each object in the `fields` array describes one credential input shown in the Settings UI:

| Key        | Type    | Description |
|------------|---------|-------------|
| `name`     | string  | Python key used to look up the value in `credentials` dict inside `plugin.py`. |
| `label`    | string  | Human-readable label shown next to the input in the Settings UI. |
| `type`     | string  | Input type: `"text"` for plain text, `"password"` for masked input. |
| `required` | boolean | Whether the field must be filled before the source can be enabled. |

### Example — keyed source

```json
{
  "source_key": "mysource",
  "display_name": "My Source",
  "description": "Job board with a free API key.",
  "home_url": "https://mysource.example.com",
  "fields": [
    {
      "name": "api_key",
      "label": "API Key",
      "type": "password",
      "required": true
    }
  ]
}
```

### Example — keyless source

```json
{
  "source_key": "mysource",
  "display_name": "My Source",
  "description": "Public API — no credentials needed.",
  "home_url": "https://mysource.example.com",
  "fields": []
}
```

---

## Notes

- Folders whose names start with `_` are skipped by the plugin loader and by
  `tests/test_source_json.py`. This folder (`_template`) is intentionally skipped.
- The loader reads only the five keys documented above. Any additional keys in
  `source.json` are ignored by validation but will be attached to `cls._plugin_schema`.
- `settings_schema()` is NOT defined in `plugin.py` — it is auto-generated by the
  loader from `source.json` and injected onto the class at load time.
